第十一课 初识DOM

DOM的核心对象就是document对象

document对象是浏览器内置的对象，里面存储着专门用来操作元素的各种方法

DOM：页面中的标签，我们通过js获取到以后，就把这个对象叫做DOM 对象

1 、获取一个元素

常规方法获取到的只是赋予选择器名的标签

像html、head、body这种不能赋值选择器的标签需要用到非常规的方法

1-1获取整个外层节点 document.documentElement

1-2获取head节点 document.head

1-3获取body节点 document.body

1-4标准获取内部节点的方法 document.getElementById(“box”) 和getElementsByClassName(“box”)

区别在于一个只能选中一个标签，另一个可以选中多个

1-5 通过标签名获取内部节点 document.getElementsByTagName(“li”),返回伪数组

和上面的方法用法一样，不同之处是tagname是选中一类标签

1-6 通过name 属性来选择内部标签（一般用于input标签）

1-7 基本通用的方法 document.querySelector 低版本浏览器有兼容性问题

6、DOM节点

DOM节点分为：元素节点、文本节点、属性节点三大类

6-1元素节点

getElementById等获取到的是元素节点

6-2属性节点

通过getAttribute获取的就是属性节点

6-3文本节点

我们通过innerText获取的就是元素的文本节点

document节点是页面根节点

html是最大的元素节点，叫做根元素节点

其他的包含在HTML内部的节点就属于元素节点

文本内容也属于一个节点，叫做文本节点

元素属性用于描述元素，叫做属性节点

注释内容也属于一个节点，叫做注释节点

6-4获取节点内容

1、id.childNodes（获取所有子节点）

2、id.children（获取所有子元素节点）

3、id.firstChild（获取第一个子节点）

4、id.firstElementChild（获取第一个子元素节点）

5、id.lastChild（获取最后一个子节点）

6、id.lastElementChild（获取最后一个子元素节点）

7、id.previousSibling（获取前一个兄弟节点）

8、id.previousElementSibling（获取前一个兄弟元素节点）

9、id.nextSibling（获取后一个兄弟节点）

10、id.nextElementSibling（获取后一个兄弟元素节点）

11、id.parentNode（获取父节点“大部分还是元素节点”）这个获取的最大节点属于document节点

12、id.parentElement（获取父元素节点）这个获取不到最大的父节点，最大就是html

属性节点不涉及父子关系getAttribute方法要拿到精准的key值才能返回相应的准确属性值

13、id.attributes [ ]拿到的是一个属性列表，可以通过索引值来拿到对应索引的属性值

6-5操作DOM节点

操作节点无非增删改查

1、创建DOM节点

var odiv = document.createElement(“div”)

odiv.innerHTML=”我是新创建的节点” //创建的节点不会显示，因为需要手动插入才可以

2、插入DOM节点

想在一个节点内部插入的话就需要获取到节点的id或者class然后

box.appendChild(odiv) //box是父节点的id

如果知道想插入位置的后面一个节点和父节点也可以通过insertBefore来向上插入

box.insertBefore(odiv,child) //box为父节点，child为后面的子节点

3、删除DOM节点

要删除子节点需要知道父节点，和需要删除的子节点

box.removeChild(child)

要删除自己的节点和后代，只需要得到自己的节点就行

box.remove()

4、替换DOM节点

需要知道新的节点和老的节点以及父节点

box.replaceChild(new,old)

5、克隆DOM节点

就是将现有的DOM节点复制一份，再进行插入操作即可

var ocloneBox = box.cloneNode()
ocloneBox.id=”box2” //由于是克隆所以id也相同，需要把id给改变一下

这个方法内部需要传一个布尔类型的参数，false就说明不克隆孩子节点，true说明克隆孩子节点

案例 动态删除

6-6节点属性

节点自带nodeType属性，不同类型的节点返回的值是一个不同的数字

nodeName属性会返回相应的属性值比如文本节点返回#text

nodeValue属性会拿到相应节点的值，元素节点因为只是一个标签，所以没有Value值

参照下表：

~ nodeType nodeName nodeValue
元素节点 1 大写标签名 null
属性节点 2 属性名 属性值
文本节点 3 #text 文本内容

6-7 获取元素尺寸

我们之前获取宽高可以通过getComputedStyle.width获取，但是这种获取方法只能获取元素内部的尺寸，一整个元素的border和padding并没有计算在内，不太具有参考价值

style.width只能获取行内的，更不靠谱，所以要介绍一种获取整个元素的占地空间的方法：

offsetWith 和 offsetHeight
  • offsetWidth ： 获取的是元素 内容 + padding + border 的宽度
  • offsetHeight ： 获取的是元素 内容 + padding + border 的高度
clientWidth 和 clientHeight

  • clientWidth ： 获取的是元素 内容 + padding 的宽度

  • clientHeight ： 获取的是元素 内容 + padding 的高度

注意:

  • 获取到的尺寸是没有单位的数字
  • 当元素在页面中不占位置的时候， 获取到的是 0
    • display: none; 元素在页面不占位
    • visibility: hidden; 元素在页面占位

6-8 获取元素偏移量

获取偏移量就是元素在页面上相对于参考 父级的左边和上面的距离

offsetParent
  • 获取元素的偏移量参考父级
  • 其实就是假设你要给一个元素 绝对定位 的时候
  • 它是根据谁来进行定位的
  • 那么这个元素的偏移量参考父级就是谁
offsetLeft 和 offsetTop
  • 获取的是元左边的偏移量和上边的偏移量
  • offsetLeft ： 该元素相对于参考父级的左侧偏移量
  • offsetTop ： 该元素相对于参考父级的上侧偏移量

这两个方法是相对于有定位的父节点说的偏移量，有定位的父级就是有position属性的的父节点

如果父节点没有position属性，则会相对body计算偏移量

clientLeft和clientTop

计算的是自己的元素相对于自己左上角的偏移量

这个值可以通过border-left-width和border-top-width改变

6-9获取可视窗口的尺寸

document.documentElement.clientWidth 和 document.documentElement.clientHeight

之前在BOM板块学过获取可视窗口尺寸的方法就是innerHeight和innerWidth，但是这两个方法会将滚动条的尺寸考虑在内

在DOM中提供了一个方法可以获取不包括滚动条的整个窗口尺寸就是document.documentElement.clientWidth和document.documentElement.clientHeight
